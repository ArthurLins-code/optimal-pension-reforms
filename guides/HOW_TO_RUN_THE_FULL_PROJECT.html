Run Guide

How to run the full project

optimal-pension-reforms — MVPF/WMVPF of Brazil's 2015 pension reform. This guide takes you from a fresh checkout to figures, tables, the WMVPF numbers, and the compiled slide deck — on the 5% sample or the full server data.

Two paths: sample · server No setwd, no path editing One command per phase

Companion: guides/RESTRUCTURE_EXPLAINER.html explains why the repo is shaped this way.

0 Quick start

Open R (or a terminal) at the repo root and run one master per phase. The environment (sample vs server) is detected automatically.

🟢 5% sample (your laptop)

# terminal, at repo root
Rscript analysis/analysis_all.R
Rscript presentation/build_deck.R

Build is skipped — the panel is pre-supplied. ~3–4 min total.

🔵 Full data (server)

# terminal, on the restricted server
export PENSION_DATA_MODE=full
Rscript build/build_all.R
Rscript analysis/analysis_all.R
Rscript presentation/build_deck.R

Rebuilds the panel from raw SUIBE/RAIS first.

Prefer scripts to the terminal? Open R/RStudio at the repo root and source(here::here("analysis","analysis_all.R")) — see §4/§5. RUN.R is a signpost that lists the entry points.

1 What it runs — the pipeline

The project moves administrative records → an eligible-worker panel → claiming counterfactuals → DiD on benefits → the policy elasticity → the MVPF/WMVPF welfare numbers → the slide deck. Three master scripts orchestrate it.

The analysis chain in detail: E4 (claiming figures) · gabriel→pure (counterfactual claim counts & pure L/S schedules) · G5 (DiD on average benefits) · I4 (WMVPF, actual reform) · I6 (WMVPF + pure decomposition + η — the headline numbers).

Data

Confidential data never enters the repo. The sample/full data live outside the repo and are gitignored; config/paths.R points at them. Never commit a CPF-bearing file.

3 Environment & config — how the data root is chosen

Every script begins with two lines and then uses one PATHS list — there is no setwd and no hardcoded path anywhere in stage code:

source(here::here("config","paths.R"))      # PROJECT_ROOT, DATA_MODE, PATHS, run_stage(), clear_dirs()
source(here::here("config","constants.R"))  # gamma, eta, replacement-rate schedules, thresholds

config/paths.R finds the repo root via the .here sentinel and resolves DATA_MODE loudly:

  1. If PENSION_DATA_MODE is set to full or sample → use it.
  2. Else, auto-detect: if the sample root exists → sample; else if a full root exists → full.
  3. Else → stop() listing the roots it looked for. Never a silent wrong path.

Environment variables (override the defaults in config/paths.R)

VariableWhat it doesDefault
PENSION_DATA_MODEforce full / sampleauto-detect by which root exists
PENSION_SAMPLE_ROOT5% sample dir (holds data/ + output/)…/OneDrive/…/transfer_may_retirement
PENSION_FULL_ROOTfull-data analysis rootF:/…/directory_2025
PENSION_FULL_BUILD_ROOTfull-data build rootU:/Documents/Paper/directory_2025
PENSION_R_LIBPATHR library path (full mode)F:/docs/R-library

Set a variable in R with Sys.setenv(PENSION_DATA_MODE="full"), or in a shell with export PENSION_DATA_MODE=full (PowerShell: $env:PENSION_DATA_MODE="full").

4 Run on the 5% sample local

The common path — validation on your machine. The panel is pre-supplied, so the build stage is skipped (build/build_all.R deliberately stop()s in sample mode).

  1. Confirm the sample root exists (or point PENSION_SAMPLE_ROOT at it). It must contain data/dt_sampled_anon.csv + data/panel_sampled_anon.csv and an output/ dir holding the prerequisite tables (see the callout).
  2. Run the analysis master — either way works: 
    # R / RStudio (open at the repo root):
source("RUN.R")                                   # optional: prints the entry points
source(here::here("analysis","analysis_all.R"))

# …or terminal:
Rscript analysis/analysis_all.R
    It runs E4 → gabriel → pure → G5 → I4 → I6 in order (≈2–3 min of compute on the 928 MB panel), writing figures + tables to the sample root's output/.
  3. Build the deck (§6): Rscript presentation/build_deck.R.
Prerequisite tables. The analysis master runs a check and stop()s if the tables it consumes are missing: output/F/F5_table_results.csv (legacy F5, read by gabriel) and output/G/G4_table_results.csv + output/H/H2_table_results.csv (and their _sample variants, read by I4/I6). On the standard sample root these are already present.

5 Run on the server full data

The real pipeline. Here you also rebuild the panel from raw SUIBE/RAIS before estimating. Set the mode once, then run all three masters in order.

# R / RStudio on the server, at the repo root:
Sys.setenv(PENSION_DATA_MODE = "full")
source(here::here("build","build_all.R"))         # raw SUIBE/RAIS → analysis-ready panel
source(here::here("analysis","analysis_all.R"))   # panel → estimates, figures, WMVPF
source(here::here("presentation","build_deck.R")) # figures → deck

# …or as a shell / batch job:
export PENSION_DATA_MODE=full
Rscript build/build_all.R
Rscript analysis/analysis_all.R
Rscript presentation/build_deck.R
From-scratch rebuild. build_all.R runs the canonical heads A4 · B4 · C6 · D4. A true raw rebuild also needs the upstream lineage that fills working/ — A1–A3, the Stata B1–B3, C1–C5, D1–D3 — run first. The Stata steps go via stata-mp -b do build/code/<file>.do. See build/code/README.md for the order and which files are canonical vs. upstream-lineage.

Note: the build root (U:/…) and the analysis root (F:/…) differ on the server — both are centralized in config/paths.R (flagged as O11 in quality_reports/restructure_findings.md).

6 Build the slide deck

presentation/build_deck.R wraps three steps; run it after the analysis master:

  1. collector.py gathers each stage's figures (newest of the sample vs repo locations) into presentation/figures_central_folder/from_code/.
  2. latexmk -cd -g -pdf compiles presentation/latex/presentation/_main.tex → _main.pdf (≈142 pages on the sample).
  3. verify_deck.py confirms every \includegraphics resolves.
Rscript presentation/build_deck.R        # collect → compile → verify
# open the result:
# presentation/latex/presentation/_main.pdf

The English deck is the live build (presentation/latex/presentation/); the Portuguese apresentacao/ deck is separate and off the collector pipeline.

8 Troubleshooting

SymptomCause & fix
could not resolve DATA_MODE …No data root found. Set PENSION_DATA_MODE and/or PENSION_SAMPLE_ROOT/PENSION_FULL_ROOT to a path that exists.
missing prerequisite table(s) … (from analysis_all.R)The legacy/sibling tables (F5, G4, H2) aren't in output/. On the sample they ship with the root; on the server run the producers first.
build_all.R needs the full server dataYou ran the build master in sample mode — it's full-only by design. The panel is an input on the sample; skip straight to analysis_all.R.
there is no package called '…'Install it: install.packages("…"). On the server, set PENSION_R_LIBPATH.
Deck build fails / missing figureRun analysis_all.R first so the figures exist, then build_deck.R. Check verify_deck.py output for the unresolved name.
A stage errors on setwd / a U:/ pathYou're likely running a non-canonical or full-only file (e.g. H3) on the sample. Use the masters; canonical sample stages have no setwd.

9 Cheat sheet

Each environment, two ways — pick whichever you prefer.

🟢 Sample (laptop) — no env var needed

R / RStudio — source the script:

source(here::here("analysis","analysis_all.R"))
source(here::here("presentation","build_deck.R"))

Command line / PowerShell — Rscript:

Rscript analysis/analysis_all.R

R / RStudio — source the script:

Sys.setenv(PENSION_DATA_MODE = "full")
source(here::here("build","build_all.R"))
source(here::here("analysis","analysis_all.R"))
source(here::here("presentation","build_deck.R"))

# bash:        export PENSION_DATA_MODE=full

Rscript build/build_all.R
Rscript analysis/analysis_all.R
Rscript presentation/build_deck.R
